# 商品价格查询系统产品需求文档

## 1. 执行摘要

### 1.1 问题陈述

企业内部商品价格信息更新不及时、缺乏统一查询界面，导致业务人员在售前查询价格效率低下，影响报价速度和准确性；同时管理层通过企微智能表格维护价格数据，数据管理困难。

### 1.2 解决方案

基于企业微信智能表格API，开发一个商品价格查询系统，通过企微自建应用搭建查询表单，实时查询智能表格数据并以JSON格式返回，前端根据查询结果展示商品价格和总价，支持PC端和移动端自适应。

### 1.3 成功标准

- **查询响应时间**：查询响应时间控制在200ms以内
- **数据准确率**：价格数据准确率达到99%以上
- **系统稳定性**：系统可用性达到99.9%
- **用户满意度**：内部员工满意度达到90%以上
- **功能完整性**：满足业务人员在售前查询价格的核心需求

## 2. 用户体验与功能

### 2.1 用户角色

| 角色   | 描述                         | 核心需求                    |
| ---- | -------------------------- | ----------------------- |
| 内部员工 | 企业内部业务人员，主要在售前阶段使用系统查询商品价格 | 快速查询商品价格、获取报价总额、移动端便捷操作 |
| 管理层  | 负责维护商品价格数据，通过企微智能表格管理      | 数据管理便捷、权限控制、数据准确性       |

### 2.2 用户故事

1. **故事1**：作为内部员工，我希望通过简单的查询表单输入商品信息，快速获取商品价格，以便及时给客户报价。
2. **故事2**：作为内部员工，我希望查询结果能清晰展示商品价格和总价，以便快速计算报价总额。
3. **故事3**：作为内部员工，我希望系统在移动端有良好的适配，以便在外出时也能随时查询价格。
4. **故事4**：作为管理层，我希望通过企微智能表格方便地维护商品价格数据，以便确保数据的及时性和准确性。
5. **故事5**：作为管理层，我希望系统有适当的权限控制，以便保护价格数据的安全性。

### 2.3 验收标准

1. **查询功能**：
   - 支持通过发电机组功率(KVA/KW)、柴油发动机型号、发电机型号等关键字查询商品价格
   - 查询响应时间不超过200ms
   - 查询结果包含所有产品信息
2. **界面体验**：
   - 界面简洁清爽，突出查询结果
   - 支持PC端和移动端自适应
   - 移动端操作便捷，适合外出使用
   - 查询结果展示清晰，易于理解
3. **数据管理**：
   - 管理层可通过企微智能表格直接维护商品价格数据
   - 数据更新后系统能实时获取最新价格
   - 支持数据的只读性查询操作
4. **权限控制**：
   - 内部员工仅可查询价格，不可修改数据
   - 管理层可修改价格数据（在项目中不用实现，企微智能表格系统来限制）
   - 确保数据访问安全

### 2.4 非目标

- 不支持在线支付功能
- 不支持复杂的数据分析功能
- 不支持与其他系统的集成（如ERP、CRM等）
- 不支持多语言切换

## 3. 技术规格

### 3.1 架构概述

- **前端**：基于企业微信自建应用的H5页面，支持PC端和移动端自适应
  - 有且仅有一个界面：通过下拉菜单或输入框生成的查询界面，查询后出报价即可
  - 进入界面时显示loading动画，待查询所有数据结束后，再进行查询界面渲染
  - 查询界面里的所有数据需要从智能表格里实时读取，不做本地或浏览器缓存，保持数据实时性
- **后端**：基于企业微信API的服务，负责与智能表格API交互
- **数据存储**：企业微信智能表格，作为关系型数据库使用

### 3.2 集成点

- **企业微信智能表格API**：用于查询、读取商品价格数据
  - 接口：`https://qyapi.weixin.qq.com/cgi-bin/wedoc/smartsheet/get_records`
  - 功能：获取智能表格中的商品记录信息
- **企业微信自建应用**：作为系统的访问入口
  - 配置：在管理端"协作-文档-API"中配置为"可调用接口的应用"
  - 权限：需要具有"文档"权限

### 3.4 安全与隐私

- **数据访问控制**：基于企业微信的权限体系，确保只有授权用户能访问系统
- **数据传输安全**：使用HTTPS协议进行数据传输
- **敏感信息保护**：价格数据仅对内部员工和管理层开放
- **API调用安全**：使用企业微信提供的access\_token进行API调用认证

## 4. 风险与路线图

### 4.1 分阶段推出计划

- **MVP阶段**：
  - 实现基础查询功能
  - 支持单商品查询
  - 移动端适配
  - 数据读取与展示
- **v1.0阶段**：
  - 支持多商品批量查询
  - 实现总价计算功能
  - 优化查询性能
  - 完善权限控制
- **v2.0阶段**：
  - 增加历史价格查询
  - 支持价格趋势分析
  - 优化用户界面
  - 增加更多查询条件

### 4.2 技术风险

- **API调用限制**：企业微信智能表格API可能存在调用频率限制，需要合理设计缓存机制
- **数据同步延迟**：智能表格数据更新后，系统可能存在短暂的数据同步延迟
- **移动端适配**：不同设备的屏幕尺寸差异可能影响移动端用户体验
- **系统稳定性**：依赖企业微信API的稳定性，可能受到外部服务影响

### 4.3 风险缓解策略

- **缓存机制**：实现本地缓存，减少API调用频率
- **数据校验**：定期校验数据一致性，确保查询结果的准确性
- **响应式设计**：采用响应式布局，确保在不同设备上的良好体验
- **错误处理**：完善错误处理机制，提高系统的容错能力

## 5. 技术实现细节

### 5.1 智能表格配置

- **创建智能表格**：通过API创建智能表格，获取docid
- **配置字段**：按照数据结构配置商品表字段
- **权限设置**：设置合适的权限，确保管理层可编辑，内部员工仅可查看

### 5.2 API调用流程

1. 获取企业微信access\_token
2. 调用智能表格get\_records接口查询商品数据
3. 解析返回的JSON数据
4. 前端处理并展示查询结果

### 5.3 前端实现

- **技术选型**：HTML5 + CSS3 + JavaScript
- **响应式设计**：使用媒体查询和弹性布局，适配不同屏幕尺寸
- **界面风格**：简洁清爽，突出查询结果
- **交互设计**：简单直观的查询表单，快速响应的查询结果展示

### 5.4 后端实现

- **技术选型**：Python + Flask
- **功能模块**：
  - access_token管理：生成后缓存7200秒，不足100秒时向企业微信请求生成新的access_token，减少API请求次数
  - API调用封装
  - 数据处理与转换
  - 错误处理与日志
  - 配置管理：读取.env文件中的corpid、corpsecret、docid、sheet_id等配置参数

## 6. 性能优化

### 6.1 前端优化

- **减少HTTP请求**：合并CSS和JavaScript文件
- **缓存静态资源**：合理设置缓存策略
- **延迟加载**：非关键资源延迟加载
- **压缩资源**：压缩HTML、CSS和JavaScript文件

### 6.2 后端优化

- **API调用优化**：合理使用API调用参数，减少数据传输量
- **缓存策略**：实现数据缓存，减少重复查询
- **并发处理**：优化并发请求处理，提高系统响应速度
- **错误重试**：实现API调用错误重试机制，提高系统稳定性

## 8. 部署与维护

### 8.1 部署方案

- **前端部署**：部署在企业微信自建应用中
- **后端部署**：部署在企业内部服务器或云服务器上
- **配置管理**：
  - 项目根目录应包含.env配置文件，由应用管理员手动配置
  - .env文件应包含corpid、corpsecret、docid、sheet_id等参数
  - 后端按配置文件里的参数创建API查询代码
  - .env文件里的参数为项目里唯一参数，所有查询都调取该配置文件，方便后期管理人员修改
